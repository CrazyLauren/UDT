Модуль FrontEnd

Модуль FrontEnd  ядра UDT  предназначен для  удобного взаимодействия  пользователя  с внешними интерфейсами и программами. Например, если  пользователю необходимо принимать данные по последовательному порту, то он может настроить модуль  на открытие последовательного порта, а если формат передаваемых данных по последовательному порту отличается от  формата, используемого в пределах системы UDT, то модуль может их конвертировать. Таким образом, пользовательское ПО становится более универсальным. 

Модуль настраивается при помощи конфигурационного файла в теге  front_end_manager. Для создания нового внешнего интерфейса необходимо добавить дочерний тег frontend, общий вид тега представлен ниже

         "frontend":{

                   "<тип интерфейса>": "<его настройки>",

                   "name":" <имя в системе UDT>",

"expl":"<выходной протокол>",

"mtu":"<mtu>",

"onerror":"<тип>",

                   "demand":[

                                      {

                                      "f":"<от кого>",

                                      "pl":"<протокол>",

                                      "num":"номер сообщения"

                                      }

                                      ]

                            }

 В теге "<name>" указывается  имя, по которому будет доступен фронтенд.

В теге "<expl >"  указывается используемый для работы с интерфейсом  протокол, если нет необходимости преобразовывать  протоколы, то тег не указывается. В текущей версии тег не поддерживается, поэтому тег указывать не надо или нужно в качестве протокола указать raw. Все данные передаются по raw протоколу.

В теге " mtu " указываются ограничения на передачу данных по интерфейсу (см. настройка ограничений на канал). Если ограничений нет, то тег не указывается.

В теге "demand" содержится информация, какие данные и от кого  отсылаются по интерфейсу.  Тег представляет собой массив содержащий объекты с полями "f","pl",  "ver", "rh" или другие.  В теге "f" указывается от кого получать пакет, в теге "pl" указывается название протокола. Далее сложнее. Если указан тег "rh" то структура  required_header_t  заполняется следующим образом: поле FVersion  берётся из тега "ver" (формат значения тега "<major>.<minor>.<revision>"), а поле FReserved берётся из тега "rh", значение которого должно быть задано в формате base64. Если тег "rh" не указан, то для заполнения структуры required_header_t  будет вызван метод MHeader протокола указанного в "pl". Если тег в теге "pl" указан протокол raw, то номер сообщения будет считан из поля «num».

Все сообщения отправляются по интерфейсу последовательно, т.е. до тех пор, пока сообщение от одной программы не будет передано, сообщения от других  программ будут буферизироваться. Если при передаче сообщения произойдёт сбой, то выбор действия зависит от  тега "onerror": "close","reopen", "ignor". По умолчанию, "onerror":"ignor".

После создания фронтенда он интерфейс будет доступен в системе UDT под именем указном в теге "name". Получение данных осуществляется обычном способом, необходимо создать «запрос» на получение данных, подробнее см. руководство. При использовании raw протокола, номер всех сообщений равен нулю.
